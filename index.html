How do we measure performance?

LCP - Largest Contentful Paint

Monitors how your website loads and finds the largest paint on your content. It finds the biggest block that has some content and measures how long it takes to paint.

INP - Interaction to Next Paint

If somebody interacts (clicks a button, adds a card, searches, etc.) how long does it take to paint? For example, if opening a card shows a spinner for 7s before going to the next page, that 7s counts as INP.

CLS - Cumulative Layout Shift

Measures how stable your UI is. Poor load can be caused by sudden image loading changing the entire page or ads coming from third-party libraries.

RAIL Model

Response

If you click something, how does your app respond? Is it fast or slow?

Animation

What is animated and how smooth is it? How fast does it run?

Idle

When doing operations, your app should be idle, not frozen, waiting to finish.

Load

How long does it take for the page or action to load?

What can Chrome DevTools Do?

Note: Chrome extensions often mess with DevTools data, leading to inaccurate reporting. If you're going to run any important audits, make sure you have all extensions disabled or use an incognito tab!

The basics of the Elements tab, Styles, and Computed are straightforward.

Elements

The full structure of your HTML. You can edit and change whatever you want.

  • Styles: See all styles applied. Use :hov to simulate different actions like hover.
  • :cls: Used for adding classes.
  • Brush icon: Set user preference for dark or light mode.
  • Computed: Shows the box model and all styles attached to the selected element.
  • Event Listeners: See events running on that exact element.

Network

Monitor network requests and analyze loading performance.

  • Files: See where files come from with detailed info on each.
  • Top bar: Customize internet connection, disable caching, etc.
  • Bottom info: Amount of requests, transferred data, load times, etc.

Performance

Focus on what the main thread is doing. For 3D or heavy animation, GPU matters.

Console

Write math, code, and execute operations on the page directly.

Sources

A full IDE environment. Note: Modern projects with bundlers may have less readable code.

Application

Access cookies, local storage, session storage, and other stored data.

Memory

Used for C, C++, and assembly to see memory usage. Also for detecting JavaScript memory leaks.

Checking Memory Usage

  1. Open Chrome's hamburger menu → More tools → Task Manager to see memory per tab
  2. Use Performance tab in DevTools with Memory profiling
  3. Use the Memory tab in DevTools:
    • Shallow Size: How big the item is
    • Retained Size: How much memory frees if removed
    Best approach: Create two snapshots and compare them

You don't get exact access to see where memory leaks happen in your code—you have to search manually.
